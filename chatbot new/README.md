# 🌐 منصة SOLIX - مهندس البيانات الافتراضي الشامل
### 🤖 SOLIX Virtual Data Engineer Assistant & RAG Chatbot

---

## 📌 1. مقدمة عامة ورؤية المشروع (Project Vision)
منصة **SOLIX** هي منصة مستقلة ومتكاملة لمهندس البيانات الافتراضي (Virtual Data Engineer) تقوم بأتمتة دورة حياة البيانات بالكامل؛ بدءاً من تجميعها وتنظيفها برمجياً داخل بيئة معزولة، ثم تدريب واختبار نماذج التعلم الآلي وتمثيل النتائج بيانياً، مع واجهة تفاعلية ذكية ودعم المعالجة فائقة السرعة.

يسعى مشروع **SOLIX** بشكل أساسي إلى تقليص الوقت المستغرق في إعداد البيانات من ساعات أو أيام إلى بضع ثوانٍ معدودة، مما يسمح لمحللي البيانات والعلماء بالتركيز المباشر على بناء النماذج واستخلاص الرؤى دون تشتيت أو تعقيد.

---

## 🎯 2. الأهداف الرئيسية للمنصة (Project Objectives)
*   **أتمتة خطوط سير البيانات (Data Pipelines):** التخلص من كتابة الأكواد المتكررة لمعالجة البيانات وتجهيزها.
*   **السرعة الفائقة:** تسريع عمليات الاستكشاف والتنظيف والتدريب الإحصائي بفضل الجمع بين محركات المعالجة الهجينة.
*   **دعم بيئة التشغيل الآمنة (Sandbox):** عزل الأكواد البرمجية لحماية الخلافات والخوادم المضيفة.
*   **التفاعل الطبيعي والدلالي:** توفير شات بوت ومساعد صوتي ذكي يفهم أسئلة المستخدمين بدقة ويوجههم بناءً على وثائق المنصة.
*   **التدريب السريع ونمذجة البيانات:** توفير بيئة AutoML متكاملة لتسهيل تدريب نماذج الذكاء الاصطناعي وتقييمها.

## ⚡ 3. الميزات والوظائف الأساسية (Core Features)

1.  **مهندس البيانات الافتراضي المستقل (Virtual Data Engineer):** إدارة دورة حياة البيانات من الرفع والتنظيف وحتى التصدير برمجياً وبشكل مؤتمت بالكامل.
2.  **محمل الملفات الشامل (Universal Loader):** سحب وإسقاط وتعرف تلقائي على صيغ الملفات المتنوعة (`Excel`, `CSV`, `JSON`, `XML`, `Parquet`, `Feather`, `ORC`) وفهرستها لحظياً.
3.  **استوديو التنظيف الذكي (Smart Data Cleaner):** مستويات تنظيف تصاعدية (`Alpha` و `Beta` و `Gamma`) لتعديل تنسيق الهواتف والتواريخ، وحساب وعزل القيم الشاذة إحصائياً، وتعديل الكلمات الخاطئة إملائياً.
4.  **محرك تعبئة الفراغات الذكي (AI Imputer Engine):** ملء الفراغات بالتنبؤ بناءً على البيانات المحيطة وخوارزميات تعلم الآلة بدلاً من تعبئة المتوسطات الحسابية الثابتة.
5.  **المساعد الصوتي "سول" بالعامية المصرية (SOL Voice Copilot):** واجهة دردشة واستعلام صوتية بلهجة مصرية تقنية، تفهم الأوامر وتولد الأكواد وتقوم بتحديث البيانات ثم تتحدث بالنتيجة.
6.  **استوديو التعلم الآلي المؤتمت (AutoML Studio):** تدريب النماذج، اختبار الدقة، ضبط المعاملات الفائقة (Hyperparameters)، ومقارنة النتائج للمطورين وتصدير الملفات الجاهزة.
7.  **مستشار البيانات الذكي (AI Dataset Advisor):** فهم نوايا استعلام المستخدم والبحث الفوري في مستودعات البيانات وعرض النتائج ونسبة ملاءمتها مع تبرير كامل بالذكاء الاصطناعي.
8.  **استوديو البيانات الاصطناعية (Synthetic Data Studio):** محاكاة البيانات الحقيقية وتوليدها بأساليب إحصائية ونمذجة متقدمة (`TVAE`, `CTGAN`, `Gaussian Copula`) لمنع التناقضات وحماية الخصوصية.
9.  **منشئ الاستمارات الذكي ومستخرج الصور (SOL Forms & OCR):** واجهة لبناء الاستمارات وجمع الإجابات تلقائياً، إلى جانب محرك OCR لاستخراج وتدقيق محتوى الفواتير والصور وجدولتها.
10. **محرك الفوضى وتخريب البيانات (Chaos Engine & Data Sabotage):** أداة اختبار تسمح بحقن نويز وتكرارات وتلف في البيانات الجدولية لاختبار قوة وموثوقية خطوط سير البيانات بالشركة.

## 🛠️ 4. البنية البرمجية والتقنيات المستخدمة (Tech Stack)

### البنية التحتية والباكيند (Backend Infrastructure)
*   **FastAPI:** إطار عمل البايثون الحديث فائق السرعة لإدارة جميع مسارات الـ APIs والاتصال غير المتزامن.
*   **Uvicorn:** خادم الويب المتزامن لخدمة وتوليد استجابات الباكيند بأقل زمن تأخير.
*   **aiosqlite & SQLAlchemy:** طبقة الاتصال غير المتزامن لإدارة وتوثيق الحسابات، وقواعد البيانات المدمجة.
*   **SQLite:** قاعدة بيانات محلية لإدارة بيانات التسجيل وجلسات العمل وإدارة الذاكرة الدلالية المؤقتة.

### محرك المعالجة والحسابات (Processing Engine)
*   **Pandas:** الأداة القياسية لتحميل البيانات في الذاكرة وإجراء التعديلات الهيكلية وتصفية الأعمدة.
*   **Polars:** محرك الحوسبة السريع المكتوب بلغة Rust، يُسخّر في الفحص المبدئي (Profiling) للملفات الكبيرة وتفادي مشكلات نفاد الذاكرة (OOM).
*   **NumPy:** معالجة العمليات الرياضية وحساب مصفوفات الارتباط والتحويلات الإحصائية الكثيفة بشكل أسرع.

### الذكاء الاصطناعي والمعالجة الدلالية (AI & NLP)
*   **Scikit-Learn:** تدريب خوارزميات الغابات العشوائية (`RandomForest`) لملء الفراغات والبيانات المفقودة بذكاء إحصائي.
*   **Qdrant:** قاعدة بيانات المتجهات المحلية لتخزين وحساب التشابه الدلالي (Cosine Similarity) للبحث الذكي.
*   **Groq Cloud (Llama 4 Scout):** معالجة عمليات التفكير وفهم وتصنيف نية المستخدم وصياغة الردود بدقة فائقة.
*   **ElevenLabs TTS:** توليف وبث الردود اللفظية بأصوات قريبة جداً من النبرة البشرية وبالعامية المصرية.
*   **Difflib:** محرك محلي لمعالجة النصوص ومطابقة الكلمات والرموز المفقودة إملائياً (Fuzzy Matching).

### الواجهة الأمامية والتصميم (Frontend)
*   **Jinja2 Templates:** محرك القوالب الديناميكي الذي يسمح للباكيند برسم الصفحات محلياً وتمرير متغيرات الجلسة للواجهات.
*   **Tailwind CSS:** تصميم وتنسيق الواجهة المظلمة الفاخرة المعتمدة على الـ Glassmorphic وعناصر النيون والـ Visualizers التفاعلية.
*   **Chart.js / Native SVG Visualizers:** رسم وتوليد التوزيعات والترابطات البيانية للمستخدم بشكل مباشر على المتصفح.

## 🔒 5. الاعتبارات الأمنية وحماية البيانات (Security Design)
*   **بيئة التشغيل المعزولة (Execution Sandbox):** تشغيل جميع نصوص وأكواد البايثون المحدثة للبيانات داخل حاوية افتراضية محظور عليها استخدام المكتبات التي تضر بنظام الملفات المضيف (مثل `os`, `sys`, `subprocess`).
*   **التأمين ضد الهجمات:** حظر تكرار طلبات تسجيل الدخول والتحقق الثنائي لرموز الـ OTP.
*   **تحديد معدلات الطلبات (Rate Limiting):** تحديد حد أقصى للطلبات (30 طلباً في الدقيقة) لمنع هجمات الحرمان من الخدمة (DoS).

## 📝 6. محرك تصحيح الأخطاء الإملائية الجديد (Spelling Correction Pipeline)
تم تزويد الشات بوت بنظام معالجة وتدقيق ذكي للمدخلات يعمل خلف الكواليس قبل البحث الدلالي أو توجيه السؤال لضمان **فهم النصوص غير المكتملة أو المشوشة**:
1.  **فلترة الدردشة السريعة (Chitchat Filtering):** تجاوز عمليات التدقيق المكلفة برمجياً عند التعرف على الكلمات الترحيبية الشائعة لضمان استجابة سريعة وفورية.
2.  **المصحح اللغوي الذكي (LLM-based Corrector):** استدعاء نموذج لغوي سريع بترددات وحرارة صفرية (`Temperature: 0.0`) لإعادة صياغة السؤال وحل التشوهات والكلمات المنقوصة مع الحفاظ على صيغة السؤال دون الإجابة عليه برمجياً.
3.  **المصحح المحلي الفوري (Local Fuzzy Matcher Fallback):** في حال حدوث انقطاع في الشبكة أو تأخر الاستجابة السحابية، يتدخل نظام محلي يعتمد على تقنية قياس المسافة النصية (`Levenshtein Distance` عبر مكتبة `difflib`) لمطابقة الكلمات غير المكتملة بقاموس المنصة الأساسي (أسماء الأعضاء، أسماء المشرفين، الأدوات، الأقسام).

## 🚀 10. طريقة التثبيت والتشغيل (Installation & Setup)

### المتطلبات المسبقة (Prerequisites)
*   تثبيت لغة البرمجة **Python 3.10+**
*   توفير مفتاح ربط لـ **Groq API Key**
*   (اختياري) ElevenLabs API Key لدعم توليد الصوت.

### خطوات التثبيت (Installation Steps)
1.  **استنساخ المشروع (Clone the Repository):**
    ```bash
    git clone https://github.com/your-username/solix-chatbot.git
    cd solix-chatbot
    ```

2.  **إنشاء بيئة عمل افتراضية (Virtual Environment):**
    ```bash
    python -m venv venv
    # لتفعيل البيئة في ويندوز:
    venv\Scripts\activate
    # لتفعيل البيئة في لينكس / ماك:
    source venv/bin/activate
    ```

3.  **تثبيت الحزم المطلوبة (Install Dependencies):**
    ```bash
    pip install -r requirements.txt
    ```
    *ملاحظة: تأكد من تثبيت مكتبات `fastapi`, `uvicorn`, `requests`, `numpy`, `scikit-learn`, `jinja2`.*

4.  **تكوين مفتاح API (Configure API Key):**
    افتح ملف `app.py` وتأكد من تحديث متغير الـ `API_KEY` بمفتاح Groq الخاص بك:
    ```python
    API_KEY = "your-groq-api-key-here"
    ```

5.  **تشغيل الخادم المحلي (Run the Server):**
    قم بتشغيل خادم uvicorn المحلي لبدء تشغيل التطبيق:
    ```bash
    uvicorn app:app --reload --host 127.0.0.1 --port 5000
    ```

6.  **تصفح التطبيق (Access the App):**
    افتح المتصفح وتوجه إلى الرابط التالي:
    [http://127.0.0.1:5000/](http://127.0.0.1:5000/)

## 💡 11. طريقة الاستخدام والتفاعل (How to Use)
*   عند فتح الواجهة التفاعلية، يمكنك كتابة أي استفسار يتعلق بالمنصة، على سبيل المثال:
    *   `مين قائد فريق العمل؟` (سيتم التعرف فورياً وتوجيهك لدور محمد شلبي).
    *   `ما هي مميزات استوديو التنظيف الذكي؟` (سيقوم نظام RAG باستخراج الجزء المناسب من دليل SOLIX).
    *   `مين المشرفين؟` (سيعرض الشات بوت تفاصيل د. سيمون وم. نجلاء).
*   يقوم الشات بوت بتخزين الإجابات مؤقتاً محلياً (Semantic Cache)، مما يعني أن تكرار نفس الأسئلة سيعطي نتائج فورية فائقة السرعة بدون استهلاك توكينز الـ API.
