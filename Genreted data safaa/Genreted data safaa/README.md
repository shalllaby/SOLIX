# 🧬 منصة SOL لتوليد البيانات الاصطناعية — SOL Platform

منصة متكاملة لتحليل البيانات، وتوليد نسخ اصطناعية (**Synthetic Data**) عالية الجودة تحاكي الخصائص الإحصائية والعلاقات المعقدة للبيانات الأصلية بشكل كامل، دون الكشف عن أي معلومات حقيقية أو حساسة للمستخدمين. تم بناء المنصة بالكامل باستخدام لغة **Python** وواجهة **Streamlit** وخادم **FastAPI**.

يُعد هذا المشروع حلاً مثالياً لاختبار الأنظمة البرمجية، وتطوير ونمذجة خوارزميات الذكاء الاصطناعي، ومشاركة مجموعات البيانات مع المطورين والشركاء الخارجيين مع الالتزام التام والكامل بمعايير حماية البيانات والخصوصية العالمية مثل **GDPR** و**HIPAA**.

---

## 🎯 فكرة المشروع وأهدافه الرئيسية

تكمن فكرة مشروع **SOL** في حل معضلة "مشاركة البيانات بأمان". في كثير من الأحيان، تحتاج جهات التطوير أو الأبحاث إلى بيانات حقيقية لاختبار الأنظمة أو تدريب النماذج، لكن القوانين الصارمة تمنع مشاركتها بسبب احتوائها على معلومات سرية أو PII (معلومات التعريف الشخصية).

تقوم منصة **SOL** بحل هذه المشكلة عبر:
1. **استيعاب البيانات الأصلية:** قراءتها وتحليل بصمتها الإحصائية وسلوك أعمدتها.
2. **عزل الحساسية:** إزالة البيانات الحقيقية واستبدالها ببيانات وهمية تبدو حقيقية تماماً (مثل استبدال اسم حقيقي بـ "أحمد العتيبي" المولد عشوائياً).
3. **تعلّم العلاقات والأنماط:** عبر خوارزميات إحصائية ونماذج تعلم عميق متقدمة، تتعلم المنصة كيف ترتبط الأعمدة ببعضها (مثل ارتباط الراتب بالخبرة والوظيفة) وتوليد بيانات تحافظ على هذا الارتباط.
4. **تصدير آمن وفوري:** تحميل البيانات في ملفات إكسل أو كود برمي، أو نشرها مباشرة كـ API حي للمطورين.

## 🔬 كيف تعمل المنصة؟ (شرح تفصيلي لكافة المكونات)

تعمل منصة **SOL** عبر نظام متعدد المراحل يعالج البيانات بدءاً من الرفع وانتهاءً بالنشر:

### 1. 🔍 مرحلة التحليل والتصنيف الدقيق (Semantic Profiling & Analysis)
بمجرد رفع الملف (CSV أو Excel)، يبدأ المحرك في فحص البيانات عبر مسارين:
*   **🤖 الاستعانة بالذكاء الاصطناعي (Groq LLM):** تتصل المنصة بنموذج `Llama 3.3` عبر Groq لإرسال عينة صغيرة من البيانات، حيث يقوم النموذج بفهم دلالات الأعمدة وتصنيفها لنوع معين (بيانات حسابية، فئات، أسماء، هواتف، بريد، عناوين، هويات).
*   **🛡️ محرك القواعد البديل (Fallback Engine):** لتجنب انقطاع الخدمة في حال تعطل الـ API أو نفاد الحصة، يحتوي المحرك على قواعد برمجية ذكية (Heuristics) تعتمد على الكلمات المفتاحية وأنماط المحتوى لتحديد الأنواع بدقة فائقة.
*   **🔑 التخزين المؤقت الذكي (Caching):** تقوم المنصة بحساب بصمة فريدة للأعمدة والقيم لمنع إعادة استدعاء الـ API لنفس الملف، مما يسرع عملية التحليل لتصبح فورية في المرات التالية.
*   **🧩 استخراج الأنماط (Pattern Sequence Extraction):** تقوم المنصة بدراسة الأعمدة التي تحتوي على تسلسلات نصية (مثل الأكواد الوظيفية أو أرقام القطع مثل `K00752.01`) وتقوم تلقائياً باستنباط النمط العام (مثال: `K#####.##`) لتوليد قيم تحاكي نفس الترتيب.

### 2. ⚙️ محركات التوليد (Generation Engines)
تتيح المنصة 4 خوارزميات مختلفة لتناسب احتياجات السرعة والدقة:
1.  **📊 الطريقة الإحصائية (Basic):** تقوم بالتحليل الإحصائي المستقل لكل عمود وتوليد قيم تتبع نفس التوزيع الإحصائي. تتميز بالسرعة الهائلة وتستخدم تقنية تعدد المسارات (**Threading**) لمعالجة الأعمدة بالتوازي.
2.  **⚡ خوارزمية Gaussian Copula:** نموذج إحصائي متقدم يدرس الارتباط المشترك بين الأعمدة دون الحاجة لتدريب شبكات عصبية، مما يمنح توليداً سريعاً جداً (خلال ثوانٍ) مع الحفاظ على العلاقات الخطية بين الأعمدة.
3.  **🧠 نموذج TVAE (Variational Autoencoder):** نموذج ذكاء اصطناعي يعتمد على المرمزات التلقائية التباينية لتعلم تمثيل منخفض الأبعاد للبيانات ثم إعادة توليدها. سريع جداً ويحقق نتائج مطابقة ممتازة.
4.  **🧠 نموذج CTGAN (Conditional GAN):** يعتمد على الشبكات التنافسية التوليدية المشروطة. تم تصميمه خصيصاً للجداول ليقوم بتعلم العلاقات والارتباطات غير الخطية والمعقدة بين الأعمدة. يأخذ وقتاً أطول في التدريب ولكنه يعطي أعلى جودة في الواقعية.
5.  **🏆 المقارنة التلقائية (Auto-Benchmark):** تقوم المنصة بتوليد نسختين (باستخدام الطريقة الإحصائية وTVAE) وتقوم تلقائياً بحساب Fidelity Score واختيار النسخة الفائزة ذات الدقة الأعلى لتقديمها للمستخدم.

### 3. 🛡️ التغطية والتغطية التلقائية للبيانات الحساسة (PII Masking)
تكتشف المنصة تلقائياً أي عمود يحتوي على بيانات شخصية حساسة مثل:
*   الأسماء الكاملة
*   عناوين البريد الإلكتروني
*   أرقام الهواتف
*   العناوين الجغرافية
*   أرقام الهويات الوطنية أو الضمان الاجتماعي (SSN)
وتقوم فوراً بعزلها واستبدالها ببيانات وهمية بالكامل مولدة عبر مكتبة `Faker` بطريقة واقعية تماماً ومطابقة للثقافة العامة دون أي صلة بالبيانات الحقيقية.

### 4. 🧪 مختبر الإعدادات المتقدمة (Advanced Settings Lab)
*   **⚖️ التوليد الموجه (Class Balancing):** لحل مشاكل انحياز البيانات (Data Bias) في نماذج التعلم الآلي، يمكن للمستخدم إجبار خوارزمية التوليد على توزيع نسب معينة بالتساوي أو تعديلها (مثل جعل نسبة المقبولين والمرفوضين متساوية 50/50).
*   **🦠 مختبر الضوضاء (Noise Lab):** لحقن قيم مفقودة (Nulls) أو قيم شاذة ومتطرفة (Outliers) بنسب محددة لاختبار متانة الأنظمة وتجهيز البيانات لظروف العالم الحقيقي القاسية.
*   **🛡️ الشروط المخصصة (Custom Constraints):** كتابة قواعد تصفية منطقية باستخدام (Pandas Query) للتأكد من توافق الصفوف المولدة مع قواعد منطق العمل (Business Logic) الخاصة بك (مثال: `Age >= 18 and Salary > 2000`).

### 5. 🎯 تقارير الجودة ومطابقة العلاقات (Fidelity & Correlation Reports)
تقدم المنصة تقارير تفصيلية ورسومات تفاعلية لقياس دقة وجودة التوليد:
*   **Fidelity Score:** اختبار Kolmogorov-Smirnov للتحقق من تطابق توزيع الأرقام، ومقارنة التكرارات النسبية للأعمدة النصية والفئوية.
*   **مصفوفة الارتباط الحرارية (Correlation Comparison):** تعرض مصفوفة العلاقات الأصلية ومصفوفة العلاقات المولدة، وتطرحهما لحساب مصفوفة الفرق، ويقوم النظام تلقائياً بتحديد مدى جودة الحفاظ على العلاقات (ممتاز، مقبول، ضعيف).
*   **مقارنات بصرية تفاعلية:** رسومات بيانية توضح الرادار الكلي للجودة، مخططات التوزيع المزدوج (Histogram)، وBox Plot للقيم الشاذة، ومخططات أعمدة فئوية مقارنة.

### 6. 🔒 تحليل الخصوصية والأمن (GDPR Privacy Score)
تستخدم المنصة مقياس **Distance to Closest Record (DCR)** للتأكد من أن البيانات المولدة ليست مجرد نسخ كربوني للبيانات الأصلية:
*   يتم قياس المسافة الإقليدية لكل صف مولد مقارنة بجميع الصفوف الأصلية (بعد ترميز وتطبيع البيانات).
*   تحديد الصفوف التي تقل مسافتها عن "حد الخطر" (Risk Threshold) وتعتبر مهددة بالتسريب.
*   إصدار نتيجة أمان كلية وتصنيف مستوى الخطر لضمان التوافق مع تشريعات GDPR.

### 7. 🌐 النشر الفوري كـ Mock API (FastAPI REST Server)
عند الانتهاء، تتيح المنصة تصدير البيانات كملف JSON مستقل، ويقوم خادم FastAPI المرفق بتمكين المطورين من تشغيل خادم محلي يخدم هذه البيانات فوراً عبر منافذ REST API مدعومة بتوثيق Swagger تفاعلي كامل.

## 🛠️ تثبيت وتشغيل المشروع

### المتطلبات الأساسية
*   تثبيت لغة **Python 3.8** أو إصدار أحدث على جهازك.

---

### 1️⃣ الخطوة الأولى: تثبيت المكتبات المطلوبة
افتح نافذة الأوامر (Terminal/PowerShell) داخل مجلد المشروع، ونفذ الأمر التالي لتثبيت جميع المكتبات اللازمة تلقائياً:

```bash
pip install -r requirements.txt
```

---

### 2️⃣ الخطوة الثانية: تشغيل واجهة Streamlit الرسومية
لتشغيل الواجهة واستخدام كافة الميزات التفاعلية، نفذ الأمر التالي:

```bash
streamlit run "App .py"
```

*ملاحظة: اسم الملف يحتوي على مسافة ("App .py"). يرجى كتابته داخل علامات تنصيص كما هو موضح أعلاه.*
سيفتح المتصفح تلقائياً على الرابط المحلي: **`http://localhost:8501`**

---

### 3️⃣ الخطوة الثالثة: تشغيل خادم الـ Mock API
إذا كنت ترغب في نشر البيانات المولدة كخدمة ويب (REST API) للمطورين:

1. قم بتوليد البيانات الاصطناعية من الواجهة الرسومية وتنزيلها بصيغة **JSON** من تبويب "البيانات المولدة والتحميل".
2. احفظ الملف باسم `mock_data.json` داخل المجلد الرئيسي للمشروع (بجانب ملف `run_api.py`).
3. شغل الخادم البرمجي باستخدام الأمر:

```bash
python run_api.py
```

*   سيعمل الخادم على الرابط: **`http://127.0.0.1:8000`**
*   للوصول للتوثيق التفاعلي واختبار الـ API مباشرة: **`http://127.0.0.1:8000/docs`**

---

### 4️⃣ الخطوة الرابعة: إعداد مساعد الذكاء الاصطناعي (اختياري)
للاستفادة من تحليل ومقترحات الـ LLM الذكية داخل المنصة:
1. قم بزيارة موقع **[console.groq.com](https://console.groq.com)** وأنشئ حساباً مجانياً.
2. انتقل إلى قسم **API Keys** واضغط على **Create API Key**.
3. انسخ المفتاح الذي يبدأ بـ `gsk_...`.
4. يمكنك تعيين المفتاح كمتغير بيئة في جهازك باسم `GROQ_API_KEY` أو نسخه مباشرة إلى التطبيق.

## 🚀 دليل الاستخدام النموذجي

1. **تحميل الملف:** ارفع ملف بياناتك الأصلية (CSV أو Excel) في خطوة 1 بالواجهة.
2. **استشارة المساعد (اختياري):** اضغط على "حلّل بياناتي الآن" للحصول على تقرير ذكي واقتراحات حول الإعدادات من مساعد Groq.
3. **تخصيص الهيكل (اختياري):** من جدول التحكم الدقيق، تحقق من نوع كل عمود (تغيير نوع، وضع حدود دنيا وقصوى، أو استبعاد أعمدة).
4. **تحديد الخيارات:** اختر عدد الصفوف، وطريقة التوليد (مثل TVAE للسرعة أو CTGAN للعلاقات المعقدة).
5. **ضبط الإعدادات المتقدمة (اختياري):** قم بضبط توازن الفئات أو حقن ضوضاء أو شروط Pandas مخصصة.
6. **بدء التوليد:** اضغط على زر **✨ Generate Synthetic Version**.
7. **التحليل والتحميل:** تصفح تقارير الجودة والخصوصية والرسوم التفاعلية، ثم قم بتنزيل البيانات (CSV/Excel/JSON) وقاموس البيانات.
8. **النشر:** انقل ملف الـ JSON وشغل خادم FastAPI لبدء مشاركة الـ API فوراً!
