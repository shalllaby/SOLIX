# 🧬 نظام SOL Data Agent (وكيل بيانات SOL)

![SOL Data Agent Concept Image](https://cdn-icons-png.flaticon.com/512/3094/3094896.png)

## 📌 الفكرة العامة
نظام **SOL Data Agent** هو منصة ذكية متكاملة لتنظيف وتحليل البيانات، مُدعمة بالذكاء الاصطناعي (AI) وتعمل بشكل ذاتي تماماً. يهدف النظام إلى سد الفجوة بين مجموعات البيانات الخام الفوضوية وغير المكتملة، وتحويلها إلى بيانات نقية جاهزة للتحليل والإنتاج، مع توفير واجهة دردشة صوتية تفاعلية مدعومة بـ **مساعد البيانات الصوتي (SOL)** الذي يفهم ويتحدث بالعامية المصرية.

---

## 🚀 المميزات الرئيسية ونظام المساعد الصوتي (SOL Voice Copilot)

### 🎙️ المساعد الصوتي SOL (Voice Copilot)
مساعد بيانات متكامل مدمج بالواجهة يتواصل بالعامية المصرية ويقوم بتنفيذ الأوامر الصوتية لتحليل وتنظيف البيانات:
1. **بوابة حارس البوابة (Gatekeeper - `refine_user_input`)**: يقوم بفحص كلام المستخدم وتحديد نيته إذا كانت دردشة عامة (`CHAT`) أو طلب لتنفيذ عملية بيانات (`COMMAND`).
2. **تنفيذ الأوامر (Command ReAct Loop)**: عند تصنيف الطلب كـ `COMMAND` يقوم المساعد بكتابة أكواد Python (باستخدام Pandas) وتشغيلها في بيئة معزولة (Sandbox) لتعديل البيانات وتحديثها.
3. **مُركّب الإجابة النهائي (Final Synthesizer)**: يأخذ مخرجات الكود ويصيغ رداً صوتياً ونصياً دافئاً ومختصراً بالعامية المصرية لا يتجاوز 25 كلمة.
4. **توليد الصوت (ElevenLabs TTS)**: تحويل الرد المكتوب إلى رد صوتي طبيعي وفوري للمستخدم.

### ⚡ محرك الاستدلال Cerebras (Cerebras Inference Integration)
تم نقل البنية التحتية للاستدعاءات اللغوية بالكامل إلى **Cerebras Inference API** مستخدمين نموذج `gpt-oss-120b` فائق السرعة لتوفير استجابة شبه لحظية وتخطي حدود الاستهلاك اليومي لمنصات أخرى مثل Groq.

### 🍃 تحسين استهلاك التوكينز (Zero-Token File Upload)
تم تحسين النظام لتقليل استهلاك الـ Tokens ومنع استهلاك الحصص اليومية:
* **نقطة وصول محلية للبيانات**: تم إنشاء endpoint محلي `GET /api/copilot/schema/{dataset_id}` يقوم بقراءة هيكل البيانات والأعمدة والأسطر محلياً باستخدام Pandas دون أي استدعاء للذكاء الاصطناعي عند رفع الملف.
* **ترحيب محلي ذكي**: يقوم الفرونت إند ببناء رسالة ترحيبية محلية بالعامية المصرية للمستخدم فور رفع الملف تحتوي على عدد الأسطر والأعمدة دون استدعاء الـ LLM.

### 🛡️ حماية واستقرار النظام (Stability & Throttling)
* **منع التعليق (Client Fast-Failure)**: تم تهيئة عميل OpenAI لتخطي المحاولات التلقائية (`max_retries=0`) لتجنب تعليق النظام لـ 50 ثانية عند حدوث حد استهلاك (429).
* **تنظيم الطلبات (Throttling)**: إدراج توقف مؤقت `await asyncio.sleep(2)` بين دورات الـ ReAct لتجنب إرسال طلبات سريعة متتالية للـ API.
* **استعادة البنية الأصلية (Original JSON Enforcement)**: تفعيل خاصية `response_format={"type": "json_object"}` في حارس البوابة لضمان استرجاع رد JSON سليم ومتناسق دائماً وتجنب الأخطاء البرمجية.

## 🧩 الأنظمة الفرعية والأدوات المدمجة (Subsystems & Tools)

1. **محمل البيانات الشامل (Universal Data Loader)**: يعالج حزمة هائلة من أنواع الملفات المختلفة بشكل تلقائي (`CSV`, `Excel`, `JSON`, `XML`, `Parquet`, `Feather`, `HDF5`, `ORC`).
2. **محرك التخطيط الدلالي الثنائي (Semantic Binary Mapping)**: يكتشف الأعمدة والقيم المنطقية من سياقها (مثل "موافق/مرفوض") ويقوم بتحويلها مباشرة إلى (0 و 1).
3. **محرك استخراج النصوص (OCR Engine)**: استخراج النصوص والبيانات من الصور والمستندات بذكاء وتحويلها لبيانات قابلة للمعالجة والتنظيف كـ DataFrame.
4. **نماذج البيانات (SOL Forms)**: نظام لإنشاء نماذج تفاعلية لجمع البيانات المباشرة من المستخدمين وتخزينها، مما يمنع المشاكل من مصدرها.
5. **معمل الذكاء الاصطناعي ومحاكي التلف (AI Lab / Corruptor)**: بيئة اختبار لإجراء تجارب ومحاكاة أعطال وتخريب مدروس للبيانات لاختبار جودة وموثوقية التنظيف.
6. **تقارير التدقيق (Audit Report)**: مراجعة وتأكيد كل تدخل قام به الذكاء الاصطناعي والتغييرات التي طرأت على البيانات لضمان الشفافية.
