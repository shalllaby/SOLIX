# ⚙️ توثيق الباكاند (Backend Documentation) — SOL Data Agent

هذا المستند يوضح الدور الفني والوظيفي لجزء الباكاند (الخادم) في مشروع **SOL Data Agent** وكيفية ترابطه مع الأجزاء الأخرى.

---

## 🚀 نظرة عامة (Overview)
الباكاند هو "المحرك" الأساسي للمشروع، تم بناؤه باستخدام **FastAPI** في لغة بايثون. يعمل كوسيط (Orchestrator) بين واجهة المستخدم (Frontend) وبين خوارزميات الذكاء الاصطناعي وقواعد البيانات.

### 🧠 المهام الأساسية (Core Responsibilities)
1.  **المعالجة الثقيلة (Heavy Processing):** تنفيذ العمليات التي تستهلك موارد كبيرة مثل تنظيف ملفات ضخمة أو استخراج نصوص عبر الـ OCR بعيداً عن المتصفح لضمان سرعة الواجهة.
2.  **إدارة البيانات (Data Lifecycle):** استقبال الملفات المرفوعة، تخزينها مؤقتاً في الذاكرة (`_store`) لمزيد من السرعة، ثم تحويلها إلى نتائج قابلة للتحميل.
3.  **محرك الذكاء الاصطناعي (AI Engine):** استدعاء خوارزميات `RandomForest` للتنبؤ بالقيم المفقودة وتصحيح الأخطاء المنطقية في البيانات.
4.  **التأمين (Security):** إدارة عملية تسجيل الدخول، تشفير كلمات المرور، وحماية المسارات (Endpoints) باستخدام الـ JWT Tokens.

---

## 🛠️ المكونات التقنية (Technical Components)

### 1. خادم الـ API (FastAPI Server)
*   **الملف الرئيسي:** `backend/main.py`.
*   **الوظيفة:** إدارة المسارات (Routes) وتوجيه الطلبات القادمة من المتصفح (مثل `/api/upload`, `/api/clean`).

### 2. محرك تنظيف البيانات (Data Cleaning Subsystem)
*   **المكونات:** `SmartDataCleaner` (في core) و `AIImputer` (في utils).
*   **الوظيفة:** تحليل البيانات وصفياً، تحديد الشذوذ (Outliers)، واستخدام تعلم الآلة لملء الفراغات.

### 3. نظام التعرف على النصوص (OCR Engine)
*   **المسار:** `backend/tools/ocr/`.
*   **الوظيفة:** معالجة ملفات الـ PDF والصور واستخراج البيانات منها كمصفوفات قابلة للتحرير.

### 3. إدارة قواعد البيانات (Database Manager)
*   **الداخلية:** `sol.db` (SQLite) لحفظ بيانات المستخدمين وجلسات العمل.
*   **الخارجية:** دعم الاتصال بقواعد بيانات `PostgreSQL`, `MySQL`, `SQLite` لاستيراد البيانات وتحديثها مباشرة (Sync).

---

## 🔗 آلية الترابط (The Connection)

*   **مع Frontend:** يتم الربط عبر طلبات `Fetch/Axios` باستخدام صيغة **JSON** لتبادل البيانات.
*   **مع Core Logic:** يتم استدعاء موديولات بايثون المحلية لمعالجة الـ DataFrame الخاص بالبيانات.
*   **مع التخزين:** يتم استخدام **SQLAlchemy** كطبقة وسيطة (ORM) للتعامل مع قاعدة البيانات دون الحاجة لكتابة أكواد SQL معقدة بشكل يدوي في كل مكان.
