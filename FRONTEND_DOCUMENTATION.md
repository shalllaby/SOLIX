# 🚀 توثيق تصميم وتطوير الواجهة الأمامية (Frontend)

هذا الملف يشرح بنية الواجهة الأمامية لنظام **SOL Data Agent**، والتطويرات التي تم تنفيذها، والتقنيات المستخدمة لتقديم تجربة مستخدم (UX) مستوحاة من الخيال العلمي (Sci-Fi) وواجهات الـ Glassmorphism التفاعلية.

---

## 🛠️ التقنيات الأساسية المستخدمة (Tech Stack)

تم بناء الواجهة الأمامية بدون الحاجة إلى إطارات عمل ثقيلة (مثل React أو Vue)، واستُخدمت بدلاً من ذلك تقنية **Server-Side Rendering (SSR)** لتكون خفيفة وسريعة التفاعل بفضل معمارية FastAPI:

1. **Jinja2 Templates**: 
   - محرك القوالب الأساسي والمدمج مع FastAPI.
   - يعتمد النظام على معمارية القوالب المتوارثة (`Template Inheritance`) حيث يعمل ملف `_layout.html` كإطار عام للوحة التحكم المركزية، وملف `_public_layout.html` للصفحات الخارجية.

2. **Tailwind CSS (CDN)**:
   - تم استخدام Tailwind CSS لكتابة تصاميم سريعة وفائقة الاستجابة (Responsive).
   - تم تخصيص الـ (Theme) وتكوين الألوان في `tailwind.config` داخل ملف HTML لإنشاء طابع الـ Dark Mode.

3. **Google Fonts**:
   - `Inter`: خط العناوين الئيسية والنصوص، يعطي طابعاً احترافياً وبسيطاً.
   - `Space Grotesk`: خط العناوين والأرقام، يعزز طابع التقنية والذكاء الاصطناعي.
   - `JetBrains Mono`: خط للأكواد البرمجية والكلمات التقنية الصغيرة.

4. **Material Symbols Outlined**:
   - حزمة أيقونات Google الحديثة لإضافة أيقونات النظام الجانبية وفي لوحات المعلومات.

5. **Vanilla JavaScript (Fetch API)**:
   - تم استخدام JavaScript الأساسي للتواصل مع مسارات الـ API الخلفية (مثل تمرير توكن المصادقة في الـ headers).

---

## 🎨 استراتيجية التصميم (Design System)

تم بناء **Design System** كامل بألوان مخصصة غير تقليدية للخروج بتصميم (SaaS Data Platform) من أعلى طراز:
- **الألوان الأساسية (Primary & Accents)**: تدرجات من الـ `Electric Cyan` و الـ `Indigo` (`#bac3ff`, `#4361ee`) التي تعطي إحساس واجهات الهاكرز والذكاء الاصطناعي.
- **طبقات الأسطح (Surfaces)**: بدلاً من استخدام لون أسود واحد، تم استخدام نظام طبقات دقيق (من `surface-container-lowest` إلى `surface-container-highest`) مثل `#111220` و `#191b28` لتعميق واجهة المستخدم (Depth).
- **التصميم الزجاجي (Glassmorphism & Glows)**: تمت إضافة حدود مضيئة من اليسار (`sidebar-glow`) وتأثيرات شفافية في عناصر الـ Navigation.

## 📂 هندسة الملفات والمجلدات

تم تقسيم القوالب بدقة داخل مجلد `frontend/templates/` إلى مسارين:
### 1. مجلد (`app/`) وهو للوحة التحكم المحمية:
- `_layout.html`: القلب النابض للواجهة، يحتوي على الشريط الجانبي (Sidebar) وتتبع الجلسات (Sessions).
- `dashboard.html`: شاشة الإحصائيات العامة للمشروع.
- `cleaning_studio.html`: واجهة استوديو تنظيف البيانات المتقدم.
- `ocr_engine.html`: واجهة استخراج النصوص عبر الذكاء الاصطناعي.
- `forms_dashboard.html`: إدارة وبناء الاستمارات الإلكترونية.
- `semantic_mapper.html`: للرسم الدلالي والمطابقة.
- `ml_advisor.html`: شاشة المستشار الآلي لخوارزميات الـ Machine Learning.
- `data_noise.html`: معمل تخريب البيانات للاختبارات و التدقيق.
- `audit_report.html`: تقارير التدقيق الكاملة وعمليات الـ AI.

### 2. مجلد (`public/`) للزوار والمصادقة:
- `landing.html`: الصفحة التعريفية للمشروع (Homepage - Landing).
- `login.html` & `register.html`: أنظمة تسجيل الدخول وإنشاء الحسابات.
- `pricing.html`: خطط أسعار المشروع (SaaS Tiers).
- `_auth_left_panel.html`: مكون جاهز (Component) يعرض إحصائيات بصرية جذابة بجانب نموذج تسجيل الدخول.

## 🔐 نظام المصادقة الداخلي للواجهة (Auth & Sessions)
تمت كتابة هندسة حماية ذكية في الفرونت اند باستخدام `localStorage` والـ `Cookies`.
- يتأكد `_layout.html` في لحظة تحميل الصفحة من صحة تسجيل دخول المستخدم (عبر دالة `verifySessionAndHydrate`).
- يقوم بجلب أول حرفين من اسم المستخدم الأصلي (Initials) وصورته الرمزية ديناميكياً لتحديث أعلى الشريط والشريط الجانبي (Hydration).

---

## 💡 الملخص والقيمة المضافة:
هذا التطوير جعل النظام يعمل كمنصة **Single Page Application (SPA) - "شبه"** بفضل سرعة استجابة الـ CSS وتحميل الصفحات، ولكنه يحتفظ بصلابة وقوة الـ SSR التي يوفرها FastAPI، مع واجهة غامقة (Dark Mode) تذهل خبراء البيانات من حيث دقة التفاصيل الألوان والتناسق.
