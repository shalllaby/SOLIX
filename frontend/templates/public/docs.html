Quick Start Guide

Welcome to SOL Data Agent — the autonomous AI data cleaning platform. This guide will get you from zero to a clean dataset in under 5 minutes.

✓ No API key required to start. Register for a free account and you get 10 free cleaning jobs/month immediately.

Installation

SOL Data Agent is a hosted SaaS — no installation required. For on-premise deployments:

Bash
# Clone & install dependencies git clone https://github.com/your-org/sol-data-agent cd sol-data-agent pip install -r requirements.txt # Set environment variables export GROQ_API_KEY="your_groq_key_here" export SECRET_KEY="your_jwt_secret" # Start the server python -m uvicorn backend.main:app --reload # → http://127.0.0.1:8000

Requirements: Python 3.10+, Groq API key (free tier available). Optional: PostgreSQL / MySQL for DB Connector features.

Authentication

SOL uses JWT (JSON Web Tokens) for authentication. Tokens are issued on login and expire after 7 days.

Obtaining a Token

HTTP
POST /api/auth/login Content-Type: application/json { "email": "user@example.com", "password": "mysecretpassword" } // Response { "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...", "token_type": "bearer" }

Using the Token

HTTP
GET /api/auth/me Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

The 4-Stage Pipeline

Every cleaning operation in SOL passes through four autonomous stages:

  • Stage 1 — Analyze: Scans schema, infers types, maps missing values and outliers.
  • Stage 2 — Strategize: AI generates a tailored cleaning plan (Alpha/Beta/Gamma) from the profile + your goal.
  • Stage 3 — Execute: Applies the strategy — imputation, outlier removal, date formatting, deduplication.
  • Stage 4 — Validate: Side-by-side diff, health score, export, and optional DB write-back.

Cleaning Studio

The Cleaning Studio is the primary interface for manual and AI-assisted data cleaning. Navigate to /app/studio after logging in.

Workflow

  • Upload a file via drag-and-drop or the file picker.
  • Optionally type a natural-language cleaning goal in the text box.
  • Select a strategy (Alpha / Beta / Gamma) and click Analyze.
  • Review the AI strategy card — each column shows the proposed action and confidence score.
  • Click Execute Cleaning when satisfied.
  • Review the diff table and download the clean file or sync to a DB.

OCR Engine

The OCR Engine converts scanned images (PNG, JPG, TIFF) and PDF documents into structured DataFrames, making them ready for SOL's cleaning pipeline.

Supports table detection — complex multi-column tables in scanned bank statements or invoices are automatically parsed into rows and columns.

ML Advisor NEW

After cleaning, the ML Advisor analyzes your dataset's features and target column characteristics to recommend the most suitable model architectures.

Output

  • Top-3 recommended algorithms with confidence scores
  • Problem type detection (classification, regression, clustering)
  • Feature importance hints
  • Skeleton Python code snippet for the top recommendation

API Reference — Cleaning

All cleaning endpoints require a Bearer token in the Authorization header.

POST /api/clean/analyze

Upload a file and receive a full metadata profile.

Request Body (multipart/form-data)

ParameterTypeDescription
fileFileThe dataset file to analyze. Supported formats listed above.
goalstringOptional. Natural language cleaning goal.
strategystringalpha | beta | gamma (default: beta)
POST /api/clean/execute

Execute a cleaning strategy on an analyzed file. Returns a cleaned file + diff report.

JSON Response
{ "status": "success", "rows_original": 12840, "rows_cleaned": 12791, "cells_fixed": 1423, "health_before": 71.4, "health_after": 99.1, "strategy_used": "beta", "download_url": "/api/clean/download/abc123" }

SQL Database Integration

Connect SOL to an existing SQL database to read tables directly as datasets and write-back cleaned data without intermediary files.

Python SDK (planned)
from sol_sdk import SOLClient client = SOLClient(api_key="sol_pk_xxxxxxxx") # Read from PostgreSQL and run Beta strategy result = client.clean.from_db( connection_string = "postgresql://user:pass@host:5432/db", table = "customers", strategy = "beta", goal = "fix missing email and phone columns" ) # Write-back to the same table result.write_back(table="customers_clean") print(f"Cleaned {result.cells_fixed} cells")

⚠ The Python SDK is currently in development. Use the REST API directly for programmatic access.
