6.8 Database Design
This chapter explains the database design of the project. The database is one of the most important components of the system because it stores the information required by the application and allows the backend to access, process, and manage this information in an organized way. The database design includes the overall structure, the main tables, the relationship between tables, and the SQL commands used to create and manage the data.
6.8.1 Database Overview
A database is a structured system used to store, organize, manage, and retrieve data efficiently. In this project, the database acts as the central storage area for users, uploaded files, and cleaning reports. It ensures that the data is saved in a clear structure and can be accessed quickly when needed by the backend.
The database is important because it allows the system to keep user information, file details, and processing results in a secure and organized form. Without a database, the system would not be able to remember users, track uploaded files, or save the results of data cleaning operations.
The project can use common relational database systems such as MySQL, PostgreSQL, or SQLite. These databases store information in tables that contain rows and columns. This structure makes it easier to manage data and connect related information together.
Data inside the system can be classified into structured and unstructured data. Structured data is arranged in tables with rows and columns, such as user names and email addresses. Unstructured data includes files such as documents, images, and other uploaded materials. The system stores file information in the database, while the actual file may be saved in a folder or storage location.
6.8.2 Tables Structure
The database is divided into several tables. Each table is responsible for storing a specific type of information. This separation helps the system stay organized, reduces data duplication, and improves performance. The main tables in this project are the User Table, Uploaded Files Table, and Cleaning Reports Table.

A database table consists of columns and rows. Columns define the type of information that will be stored, while rows represent the actual records. For example, in the User Table, each row represents one user, and each column represents a detail about that user.
A Primary Key is used to uniquely identify each record inside a table. For example, the user_id in the User Table identifies each user separately. A Foreign Key is used to connect one table with another. For example, the Uploaded Files Table can use user_id as a Foreign Key to show which user uploaded each file.

6.8.6 SQL Code Explanation
SQL, which stands for Structured Query Language, is used to create, manage, and manipulate databases. It allows the developer to create tables, insert data, retrieve records, update information, and delete data when needed.
The most common SQL commands used in this project include CREATE TABLE, INSERT INTO, SELECT, UPDATE, and DELETE. Each command has a specific function in managing the database.
CREATE TABLE users (    user_id INTEGER PRIMARY KEY,    username VARCHAR(100) NOT NULL,    email VARCHAR(150) UNIQUE NOT NULL,    password VARCHAR(255) NOT NULL,    created_at DATETIME DEFAULT CURRENT_TIMESTAMP);

The CREATE TABLE command is used to create a new table in the database. In this example, the users table is created with fields for user ID, username, email, password, and account creation date.
INSERT INTO users (username, email, password)VALUES ('Ahmed', 'ahmed@example.com', 'encrypted_password');

The INSERT INTO command is used to add new records to a table. In this example, a new user is added to the users table.
SELECT * FROM users;



The SELECT command is used to retrieve data from the database. The star symbol means that all columns will be displayed.
UPDATE usersSET email = 'newemail@example.com'WHERE user_id = 1;

The UPDATE command is used to modify existing data in the table. The WHERE condition is important because it identifies which record should be updated.
DELETE FROM usersWHERE user_id = 1;

The DELETE command is used to remove data from the table. It should be used carefully because deleted data may not be recovered easily.
